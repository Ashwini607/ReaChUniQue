Documentation

What it does?


Tool has been created keeping in mind to help the researcher who is looking for getting the information through federated triple stores and understanding the schema of triple stores. Most of the information has been taken from ChEMBL triple store, a small drug like molecules store and two other triple stores used are UniProt for proteins and their associated disease and Reactome for pathway.

When you select the options from form according to your problem and submit then it generates the background SPARQL query that runs on ChEMBL SPARQL endpoint and provides the results (Limited to 11). It also provides the generated SPARQL query.

As the query execute on ChEMBL SPARQL endpoint so it has same behavior and limitation as endpoint. You can see the more information about this on EBI RDF platform web page. Link is mentioned in footer of this page.

Brief description of used terminology


ChEMBL-id: An identity provided to most of the compounds and unique in ChEMBL triple store.

altLabel: Alternative name of compound. This contains most of the trade names for a single compound.

Highest Development Phase: The highest phase the molecule has reached in drug approval process (4 = approved).

Substance Type: States the type of compound either that is small molecule or protein or cell-line etc.

Standard Type: Standardised version of the publishedType, e.g. IC50 rather than Ic-50/Ic50/ic50/ic-50.

Standard Value: Standardised version of the publishedValue. Transformed to common units: e.g. All concentration values are converted to nannoMolar, all time values are converted to minutes etc.

Standard Relation: Standardised version of the publishedRelation and is a symbol constraining the activity value (e.g. >, <, =).

Standard Unit: Standardised version of the publishedUnits, e.g. concentrations are in nM.

Assay Label: A unique identity provided to assay.

Assay Desc: Gives description about the assay.

Assay Type: Assay classification, e.g. Binding assay, ADME assay or Functional assay.

targetConfDesc: Description of the target types assigned with each targetConfScore.

targetConfScore: Confidence score, indicating how accurately the assigned target(s) represents the actually assay target. 0 = un-curated/unassigned, 1 = low confidence to 9 = high confidence.

targetRelType: Flag indicating of the relationship between the reported target in the source document and the assigned target from TARGET_DICTIONARY.

targetRelDesc: Description of targetRelType flag.

Target Label: Labeling of target.

Target Title: Title provided to target.

Target Type: Describes whether target is a protein, an organism, a tissue etc.

Organism: Source organism of molecular target or tissue, or the target organism if compound activity is reported in an organism rather than a protein or tissue.

How to operate?


Form has five boxes to select the options along with three spaces to write text

Different text colour in form signifies information taken from respective triple store.
Black for ChEMBL
Blue for UniProt
Red for Reactome

Form has multiple selection options over all boxes at a time. First box options are interlinked so if you are interested in finding all the targets corresponding to all the substances then you would have to select the intermediate option too, i.e. Activity and Assay.

Apart from first box you have freedom to choose the options whatever you want.

If you are interested in protein associated with disease then select just the disease option not protein and it will give the both information.

Spaces are for filtering your results. In first space you can write the activity standard unit such as ‘nm’, second space is for writing the activity value with relational operator. For example, if you are interested in activity less than 10 then write it like ‘<10’, and third space is for writing organism name such as Home Sapiens, Mycobacterium tuberculosis.

Reset button is there so you can reset the options if you need.

Problem: Retrieve the ChEMBl molecules targeting Mycobacterium organism. Where Standard activity unit contains 'nm', and standard activity value less than 10.

Way to solve it: You would have to select the following options Substance, Activity, Assay, Target, ChEMBL-id, Standard Value, Standard Unit and Organism and then in space below Standard Unit write 'nm', below Standard value '<10' and below Organism 'mycobacterium'.
In above snap shot, selected options have gray colour background and values written in spaces. space is a part of filtering the results so will extract the information related to all the matching text. If you are accurate with your search then you can remove the variable name from the generated SPARQL query and replace with the exact value (take care of case sensitivity). For example here we have search the unit 'nm' so it will extract the units containing 'nm, such as nM, nmol/min etc. If you are interested in just 'nM' then remove the variable ?stdUnit with 'nM' (?activity cco:standardUnits 'nM'.).

Note: Text inside space goes under FILTER construct of SPARQL query that causes longer execution time. If you know the exact value to filter then in generated query replace that with the variable name and run on ChEMBL SPARQL endpoint. For example, if you want to search results which contains activity standard unit "nm" then in generated SPARQL query replace the variable ?stdUnit from WHERE clause with ‘nM’ and remove the variable name from SELECT clause. After that run it on ChEMBL Sparql endpoint available at http://www.ebi.ac.uk/rdf/services/chembl/sparql.

Who will get benefit?


Researcher who has not or small knowledge of SPARQL query language and are unfamiliar with schema of triple stores.

What are the benefits?


Help to understand the schema of triple store (mainly ChEMBL).

No need to learn SPARQL query language for basic problem but if you want to solve the advance problem then it will help you by giving the part of SPARQL query.

Able to answer your basic problem, but the number of results will be limited to 11 which will be helpful to understand that are you getting the part of results that you were looking for or not.

How do we look to this tool?


Consider there is a tree having varieties of fruits i.e unripe to ripe and small to large. As we know It is very difficult to find the choice of all the people. In case someone is interested to serve the fruits to people according to their choice then the best thing would be give the tools to them so that with some efforts they will be able to pluck the fruits of their own choice. Similarly here we are unable to answers all the biological problems related to used triple stores so we have build a platform that could help the researcher to find the solution of their problem in bit easy way.
