    • What does this tool do?


    This tool can help the researcher to retrieve the information through the federated triple stores, and to understand the schema. Most of the information comes from ChEMBL triple store, a small drug like molecules store. Two other triple stores used are UniProt for the proteins and their associated disease and Reactome for the pathways.

    This tool has a form which allows to select the options according to your query related to compounds and their target and then submit. After submission, it generates the background SPARQL query that runs on the ChEMBL SPARQL endpoint and provides the results along with the generated SPARQL query.

    Once you submit the selected options along with the texts in the fields according to your query, the results will produce in the new page. The limit of the number of results is 11. From this many results, you can decide that the generated SPARQL query is enough for your problem or need modification. Modify the query if needed. And finally execute the query on ChEMBL SPARQL endpoint to produce more results. Look at limitation of ChEMBL SPARQL endpoint.

    Why is this tool better than using directly the ChEMBL endpoint?


    This tool is more useful for the one who is not or less familiar with the triple store and a SPARQL query language. If you go to the ChEMBL SPARQL endpoint directly , besides the knowledge of the SPARQL query language, you would have to understand the schema of the triple store. If you go to understand the schema and learn the SPARQL, can take a long time where this tool can help you. Here, you have to select the options and write the text in the fields that anyone can do. If not the complete solution, part of the solution and bit modification can help to solve your query completely.

    Description of the terminology used in this tool.


    ChEMBL-id: An identity provided to the most of the compounds and unique in ChEMBL triple store.

    Alternate Name: Alternative name of the compound, i.e. trade names.

    Highest Development Phase: The highest phase the molecule has reached in the drug approval process (4 is approved, others are 0, 1,2 and 3).

    Substance Type: States the type of compound either that is small molecule or protein or cell-line etc.

    Standard Type: Standardised version of the activity type, e.g. IC50 rather than Ic-50/Ic50/ic50/ic-50.

    Standard Value: Standardised version of the activity value. Transformed to common units: e.g. all concentration values are converted to nanoMolar (nM), all time values are converted to minutes etc.

    Standard Relation: Standardised version of the activity relation and is a symbol constraining the activity value (e.g. >, <, =).

    Standard Unit: Standardised version of the activity units, e.g. concentrations are in nM.

    Assay Label: A unique identity provided to the assay.

    Assay Description: Gives description about the assay.

    Assay Type: Assay classification, e.g. Binding assay, ADME assay or Functional assay.

    Target Confidence Score: Confidence score, indicating how accurately the assigned target(s) represents the actually assay target. 0 = un-curated/unassigned, 1 = low confidence to 9 = high confidence.

    Target Confidence Description: Description of the target types assigned with each Target Confidence Score.

    Target Relation Type: Flag indicating of the relationship between the reported target in the source document and the assigned target.

    Target Relation Description: Description about Target Relation Type.

    Target Label: Labeling of the target.

    Target Title: Title provided to the target.

    Target Type: Describes whether target is a protein, an organism, a tissue etc.

    Organism: Source organism of molecular target or tissue, or the target organism if compound activity is reported in an organism rather than a protein or tissue.

    Guide to use this tool?


    The form has seven boxes to select the options along with the fields to write the text.

    Different text colour in the form signifies that information has taken from the respective triple store.
    Black for the ChEMBL
    Blue for the UniProt
    Red for the Reactome

    The form has multiple selection options over all the boxes at a time. Box: 1's options are interlinked. If you want to retrieve all the targets corresponding to all the substances then you would have to select the intermediate options too, i.e. Activity and Assay.

    Apart from the first box, you have the freedom to choose the options whatever you want.

    If you want to filter the results by writing the text in the fields then make sure you have selected the corresponding option too. For example, if you are targeting the Homo Sapiens organism then beside writing the organism name in the field below the organism, you would have to select the Organism option too.

    The fields are for filtering your results. In the field of Published Date, Highest Development phase, Standard Value and TragetConfidence Score, write a digit along with the operator (< or > or =). For example, if you want the activity standard value less than 10 then in the field of Standard Value writes ‘< 10’. In the field of Journal Name, Standard Unit, Assay Type and Organism, write the appropriate text.

    The Reset button is there so you can reset the options if you need.

    Example: Retrieve the ChEMBl molecules targeting Mycobacterium organism where activity standard unit contains 'nm' and activity standard value less than 10.

    Way to solve it: You would have to select the following options: Substance, Activity, Assay, Target, ChEMBL-id, Standard Value, Standard Unit and Organism, and then in the field below Standard Unit, Standard value and organism write 'nm','<10' and 'mycobacterium' respectively.

Now, remove the FILTER line. See the results live

    Note: For the one who is trying the SPARQL query on the endpoint. Text inside field goes under FILTER construct of a SPARQL query that causes longer execution time. If you know the exact value to filter, from generated query remove the FILTER line and replace the variable name from remaining query. Now, execute the query on ChEMBL SPARQL endpoint. For example, if you want to search results which contains activity standard unit "nm" then in generated SPARQL query remove the FILTER line (?activity cco:standardUnit ?stdUnit) and replace the variable ?stdUnit from WHERE clause with ‘nM’. Remove the variable name from SELECT clause. After that executes it on ChEMBL Sparql endpoint.

    What if result is coming null?


    There are two reasons for the result null.

    1. Data is not present in the triple store for your query.

    2. You have filled the form incorrectly.

    There are three things to care during filling the form.
    a. You have connected the sources of box 1 by selecting the in between options too.
    b. You have selected the option for the field to fill up. For example, to filter the results for the Mycobacterium organism beside writing text in the field, you would have to select the Organism option too.
    c. You have used the operator (< or > or =) with a digit in filling the field (Published Date, Highest Development phase, Standard Value and TragetConfidence Score).

    An analogy to understand this tool.


    Consider there is a tree having varieties of fruits, i.e. unripe to ripe and small to large. As we know, the choice to eat the same fruit varies w.r.t. the people according to the varieties. It is very difficult to find the choice of all the people. In case, someone wants to serve the fruits to the people according to their choice. The best idea would be given the tools to them. The tool will help the people to pluck the fruits of their own choice. Similarly, here we are unable to answer all the biological problems related to the triple stores, therefore, we have built a platform that could help the researcher to find the solution of their problem in a bit easier way.
