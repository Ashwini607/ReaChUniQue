Documentation

What it does?


Tool has been created by keeping in mind to help the researcher to get the information through federated triple stores using semantic technology. Main information has been taken from ChEMBL triple store, a small drug like molecule store and two others triple store used are UniProt for proteins and their associated disease and Reactome for pathway.

When you select the options from form according to your problem and submit then it generates the background SPARQL query that runs on ChEMBL SPARQL endpoint. It will reproduce just a limited number of results along with SPARQL query.

Brief description of used terminology


ChEMBL-id: An identity provided to most of the compounds and unique in ChEMBL triple store.

altLabel: Alternative name of compound. This contains most of the trade names for a single compound.

Highest Development Phase: The highest phase the molecule has reached in drug approval process (4 = approved).

Substance Type: States the type of compound either that is small molecule or protein or cell-line etc.

Standard Type: Standardised version of the publishedType, e.g. IC50 rather than Ic-50/Ic50/ic50/ic-50.

Standard Value: Standardised version of the publishedValue. Transformed to common units: e.g. All concentration values are converted to values in nannoMolar, all time values are converted to minutes etc.

Standard Relation: Standardised version of the publishedRelation and is a symbol constraining the activity value (e.g. >, <, =).

Standard Unit: Standardised version of the publishedUnits, e.g. concentrations are in nM.

Assay Label: An unique identity provided to assay.

Assay Desc: Gives description about the assay.

Assay Type: Assay classification, e.g. Binding assay, ADME assay or Functional assay.

targetConfDesc: Description of the target types assigned with each targetConfScore.

targetConfScore: Confidence score, indicating how accurately the assigned target(s) represents the actually assay target. Foreign key to CONFIDENCE_SCORE table. 0 means uncurated/unassigned, 1 = low confidence to 9 = high confidence.

targetRelType: Flag indicating of the relationship between the reported target in the source document and the assigned target from TARGET_DICTIONARY. Foreign key to RELATIONSHIP_TYPE table.

targetRelDesc: Description of targetRelType flags.

Target Label: Labeling of target.

Target Title: Title provided to target.

Target Type: Describes whether target is a protein, an organism, a tissue etc.

Organism: Source organism of molecuar target or tissue, or the target organism if compound activity is reported in an organism rather than a protein or tissue.

How to operate?


In form, it has five boxes along with spaces to write text to get the specific result.

Different colour text signifies information taken from respective triple store. Black for ChEMBL Blue for UniProt Red for Reactome

Form has multiple selection options over all boxes at a time. First box options are interlinked so if you are interested in finding all the targets corresponding all the substances then you would have to select the intermediate option too that is activity and assay.

Apart from first box you have freedom to choose the option whatever you want.

If you are interested in protein associated with disease then just select the disease option not protein.

Tool have provided three text boxes for your specific enquiry. In first box you can select the activity standard unit such as ‘nm’, second box is for selecting the activity value with relational operator such as if you are interested in activity less than 10 then write it like ‘<10’, and third box is for selecting organism such as home sapiens, mycobacterium tuberculosis.

Reset button is there so you can reset the options if you need.

Problem: If you are interested in finding the ChEMBl molecules targeting Mycobacterium having standard activity unit contains nM and value less than 10.

Way to solve it: You would have to select the following options Substance, Activity, Assay, Target, ChEMBL-id, Standard Value, Standard Unit and Organism and then in text box below Standard Unit write 'nm', below Standard value '<10' and below Organism 'mycobacterium'.
Slected options have been shown in gray background and values written in text boxes. Text box is a part of filtering so will extract the information related to all the matching text. If you are accurate with your search with case sensitive then you can remove the variable name from the generated SPARQL query and replace with the value. For example here we have search the unit is nm so it will extract the unit containing nm like nM, nmol/min etc. If you are interested in just nM then remove the variable ?stdUnit with 'nM' (?activity cco:standardUnits 'nM'.).

Note: Text inside box goes under FILTER construct of SPARQL query that cause more execution time. If you are confirm about the filtering option in correct form then you can replace that. For example if you want search corresponding to nm Standard Unit then in generated code replace the ?stdUnit with ‘nM’ and run it on ChEMBL Sparql endpoint available at http://www.ebi.ac.uk/rdf/services/chembl/sparql.

Who will get benefit?


Researcher who has not or small knowledge of SPARQL query language and are unfamiliar with schema of triple stores.

What are the benefits?


Help to understand the schema of triple store (mainly ChEMBL).

No need to learn SPARQL query language for basic problem but if you want to solve the advance problem then it will help you by giving the part of SPARQL query.

Able to answer your basic problem, but the number of results will be limited to 11 which will help you to understand that are you getting the part of results that you were looking for or not.
